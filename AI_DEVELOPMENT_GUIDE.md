# AI開発ガイドライン (Supabase連携・自動化フロー)

このプロジェクトでAIがデータベース（Supabase）を伴う新機能を実装する際は、外部のMCPツールに頼らず、以下のターミナルコマンドと手法を用いて**完全自動**でDB設計からコード実装までを行うこと。

## 1. DBスキーマの確認
新しいテーブルを作成する際や既存のテーブルを操作する前は、現在のスキーマと型定義を確認すること。
- **既存のマイグレーション確認**: `supabase/migrations/` フォルダ内の `.sql` ファイルを参照する。
- **ターミナルでのクエリ実行**: `npx supabase db query "クエリ"` を用いて現在の実DBの状態を確認する。

## 2. 新規テーブル・カラムの追加 (Migrationの作成)
Supabaseダッシュボード（UI）から手動でテーブルを作成するのではなく、**必ずマイグレーションファイルを通して**定義を追加すること。
1. `supabase/migrations/` ディレクトリ内に、タイムスタンプを付けたSQLファイルを作成する（例: `20240418000000_add_new_feature.sql`）。
2. そのSQLファイルに `CREATE TABLE` や `ALTER TABLE` などの正しい定義を記述する。
3. （必要に応じて）初期データやRLS（Row Level Security）ポリシーなどの設定もあわせて記述する。

## 3. データベースへの反映 (Push)
マイグレーションファイルの作成が完了したら、ターミナルから以下のコマンドを実行してリモートDBまたはローカルDBに反映させること。
```bash
npx supabase db push
```
※ローカル環境の場合は `npx supabase migration up` など適切なコマンドを使用することも検討してください。

## 4. TypeScriptの型（Types）の自動生成
スキーマの変更が確実にDBへ反映されたら、直ちにNext.js等のフロントエンド・バックエンド用の型ファイルを更新すること。
```bash
npx supabase gen types typescript --project-id "[PROJECT_ID_HERE]" --schema public > src/types/supabase.ts
```
※プロジェクトの構成に応じて、型ファイルの出力先やCLIコマンドは適切に調整してください。

## 5. アプリケーションコードの実装
データベース側の準備および型の更新が完了した後、アプリケーション側（Next.jsのServer ActionsやAPIルート、コンポーネント）の実装を行うこと。
- Supabaseクライアント (`@supabase/supabase-js`) を用いて、新しく作成したテーブルに対して安全にアクセスするコードを構築する。
- 自動生成された型を使用して、コンパイルエラーを防ぐこと。
